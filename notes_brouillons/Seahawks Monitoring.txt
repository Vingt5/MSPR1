documentation : 

Pour le transfert de fichier de la machine nester vers la machine harvester, nous utilisons scp pour transferer les fichier. 
avec SCP, les fichier sont chiffrés lors de l'envoie. 

1. installation de d'OpenSSH sur les deux machines 
2. nester recupere le fichier et l'envoie a Harvester grave à la commande : scp test.txt user@10.251.120.230:/home/user/
3. lorsqu'on envoie un fichier avec le meme nom qu'un fichier avec le meme nom sur le repertoire distant, celui-ci est ecrassé et remplacer par la nouvelle version

Pour sécuriser l'envoie et la reception, nous remplaçons le port 22 pour un autre aléatoirement dans le fichier "/etc/ssh/sshd_config"

Nginx est utilisé en tant que serveur reverse proxy. Nginx redirige les requêtes HTTP vers notre application Node.js qui tourne en arrière-plan.

POUR LE DASHBOARD: 
installation de node.js et npm. initialisation de la config de nginx pour afficher le dashboard dans /var/www/dashboard/public/

On utilise pm2 pour :
- lancer l'environnement node au démarrage de la VM
- garder l'application toujours en ligne
- redémarre automatiquement l'application si elle plante ou si le serveur redémarre
- faciliter la gestion et le monitoring.
